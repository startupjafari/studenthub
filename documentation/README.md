# StudentHub - Документация

StudentHub - это образовательная платформа для студентов, преподавателей и администраторов университетов. Платформа предоставляет социальные функции, управление группами, обмен документами, события и многое другое.

## Структура проекта

Проект состоит из двух основных частей:

- **Backend** (studenthub-backend) - NestJS API сервер
- **Frontend** (studenthub-frontend) - Next.js веб-приложение

## Быстрый старт

### Требования

- Node.js 18+ и npm
- PostgreSQL 14+
- Redis 7+
- Docker и Docker Compose (опционально)

### Установка

1. Клонируйте репозиторий
2. Установите зависимости для backend и frontend
3. Настройте переменные окружения
4. Запустите базу данных через Docker Compose
5. Выполните миграции базы данных
6. Запустите серверы

Подробные инструкции смотрите в соответствующих разделах документации.

## Основные функции

### Аутентификация и авторизация

- Регистрация и вход пользователей
- Email верификация
- Двухфакторная аутентификация (2FA)
- JWT токены (access и refresh)
- Роли пользователей (STUDENT, TEACHER, UNIVERSITY_ADMIN, SUPER_ADMIN)

### Социальные функции

- Посты с изображениями и текстом
- Комментарии к постам
- Реакции (лайки)
- Истории (Stories) на 24 часа
- Система друзей
- Приватные сообщения
- Групповые чаты

### Управление группами

- Создание учебных групп
- Управление участниками
- Групповые сообщения
- Документы группы

### Университеты

- Управление университетами
- Назначение администраторов
- События университета
- Документы и верификация

### Уведомления

- Real-time уведомления через WebSocket
- Email уведомления
- Push уведомления (в разработке)

## Технологии

### Backend

- NestJS 10
- TypeScript
- PostgreSQL (Prisma ORM)
- Redis (кэширование и очереди)
- Socket.io (WebSocket)
- JWT аутентификация
- Swagger (API документация)

### Frontend

- Next.js 14 (App Router)
- TypeScript
- Redux Toolkit
- React Hook Form + Zod
- CSS Modules
- Socket.io Client

## Безопасность

- Helmet.js для защиты заголовков
- CORS настройка
- Rate limiting
- Валидация входных данных
- Хеширование паролей (bcrypt)
- JWT токены с истечением срока действия
